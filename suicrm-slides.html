Encryption
Encryption

Your data encrypted before it
ever leaves your device

Encrypt First

Files and notes are encrypted client-side using Seal SDK before upload to Walrus. The server never sees plaintext.

Key Sharding

The encryption key is split across N independent Seal key servers. No single server holds the full key.

Role-Gated Decrypt

Key servers only reconstruct the key after calling seal_approve in the Move contract — which checks the caller's on-chain role level.

Access Control
Access Control

Enforced on-chain,
not by a config file

Your OrgAccessRegistry Move object stores each member's role as a u8. seal_approve checks this value before authorizing decryption.

Viewer Level 1 Can decrypt resources with access_level ≤ 1. Read-only access to shared notes and files.
Manager Level 2 All viewer permissions + upload encrypted files + create notes tagged Viewer or Manager.
Admin Level 3 Full control: org creation, member invite, decrypt all resources, set any access level.

Role levels are immutable on-chain values in your OrgAccessRegistry object.
